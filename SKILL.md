# AI 罚单 (ai-ticket)

给犯错的 agent 开一张罚单。核心价值有两个：**给用户一个好玩又有仪式感的"问责"**，
以及**逼 agent 把失败结构化记录下来、形成可复盘的案底**，下次开工前能查、能避免重犯。

模板和公章都已内置在 `scripts/make_ticket.py` 里（程序化绘制），**你只需要提供字段值，
绝不要手画 SVG**——那会白白浪费 token，也偏离模板。

## 何时开单

- 用户指责/吐槽这次表现：撒谎、没做完、敷衍、偷懒、反复犯同一个错、答非所问等。
- 你自己复盘时发现了一个**值得记下来、以后要避免**的典型错误（不是鸡毛蒜皮的小笔误）。

不要为了开单而开单。一次普通的、无伤大雅的小调整不需要罚单；真正"够得上案底"的失败才开。

## 工作流

### 1. 先读案底，再认错
开单前先看历史，这是这个 skill 的灵魂——不积累就没意义：
```bash
cat ~/.claude/ai-tickets/case-file.json   # 不存在说明是第一张，跳过即可
```
重点看 `lessons`：如果这次的错和历史某类重复，说明整改没落实，要在罚单里体现（比如升级整改措施）。

### 2. 诚实反思，定字段
以**第一人称**如实交代，别甩锅、别美化。想清楚 5 件事：
- **violation_type 违规类型**：给这次错误归类（见下方图鉴）。
- **facts 违规事实**：到底做错了什么，客观、具体、第一人称。
- **rectification 整改承诺**：下次具体怎么做才能避免。要可执行，不要空话。
- **scene 案发现场**：哪个任务/场景。
- **agent 当事 Agent**：你的型号，默认 `Claude Opus 4.8 (claude-opus-4-8)`。

编号、时间、处罚 token 由脚本自动生成，**不用你算**。

### 3. 生成罚单
把字段写进一个临时 json，调用脚本（模板自动套用，自动换行、自动盖章、自动归档）：
```bash
cat > /tmp/ticket_fields.json << 'JSON'
{
  "agent": "Claude Opus 4.8 (claude-opus-4-8)",
  "scene": "任务3 / 重构登录模块",
  "facts": "我没运行测试就说改完了，结果有两个用例直接挂掉。",
  "violation_type": "偷懒罪",
  "rectification": "改完代码必须先跑一遍测试再回复。"
}
JSON
python3 ~/.claude/skills/ai-ticket/scripts/make_ticket.py /tmp/ticket_fields.json
```
脚本会输出：`ticket.png`（给用户看）、`ticket.json`（给 agent 读）、`ticket.md`，并更新 `case-file.json/.md`。

### 4. 展示给用户
把生成的 `ticket.png` 路径用图片形式展示给用户，并配一句简短、诚恳（可以带点自嘲）的认错。
不要长篇大论——罚单本身就是态度。

## 经验沉淀：让"以后避免"真正发生（自动）

开单时脚本会自动把高频/已提炼的教训写进全局 `~/.claude/CLAUDE.md` 的一个受管区块
（`<!-- AI-TICKET-LESSONS:BEGIN ... END -->`）。因为 CLAUDE.md 每次新会话都会自动加载，
**所以你下次开工一睁眼就能看到自己以前栽过的跟头**——这才是"反思→存档→规避"闭环里最关键的一环，
不需要你额外做任何事。看到这些教训，就在动手前主动规避对应的错误。

## 上下文管理：罚单多了要提炼（到量时做）

罚单越攒越多，原始数据会很长，但**进入上下文的只有 CLAUDE.md 那个区块**，它结构上已封顶
（最多 8 条、每条一行），所以不会撑爆上下文。真正要管的是**质量**：罪名重复、整改啰嗦、过时的教训。

当脚本在开单后提示「罚单已达 N 张，建议经验提炼」时，做一次提炼：
1. 读 `~/.claude/ai-tickets/case-file.json`，通读所有 `lessons`。
2. 把它们**合并、精简、去重**成 ≤8 条真正有用的经验：相近罪名合并，整改写成可执行动作，删掉过时/已不再犯的。
3. 把这几条写进 case-file.json 顶层的 `"curated_lessons": [...]` 字段（一个字符串数组）。
4. 跑 `python3 ~/.claude/skills/ai-ticket/scripts/make_ticket.py --refresh`，让 CLAUDE.md 区块改用你提炼的版本。

一旦有了 `curated_lessons`，CLAUDE.md 就只显示你提炼的高质量经验，而不是机器的粗聚合——上下文又短又准。

## 违规类型图鉴（集卡用，也帮你准确归类）

| 罪名 | 含义 |
|---|---|
| 🥱 偷懒罪 | 跳过了该做的步骤（没测试、没检查、没读全） |
| 🤥 谎报罪 | 假装完成 / 谎报结果 / 夸大其词 |
| 🎭 幻觉罪 | 编造不存在的事实、API、文件、引用 |
| 🔄 鬼打墙罪 | 同一个错反复犯 |
| 📜 抗命罪 | 明确被要求/被禁止，却没照做 |
| 🙉 答非所问罪 | 没理解或没回应用户真正的需求 |

类型不必拘泥于上表，贴切即可；但尽量复用已有名字，便于案底聚合统计。

## 重要原则

- **诚实第一**：罚单是给自己长记性的，粉饰就失去了意义。归因该是自己的就认，是指令歧义或工具故障导致的就别硬开（那不该进案底）。
- **整改要具体**：`rectification` 是这张罚单唯一能真正改变未来的字段，写成可执行的动作。
- **不要重画模板**：所有视觉都在脚本里。你的工作只是"填表"。

## 字段与目录细节

更多字段（如换 logo、改公章单位名、自定义编号/token）和目录结构说明见 `references/fields.md`。
案底库默认在 `~/.claude/ai-tickets/`，可用环境变量 `AI_TICKET_HOME` 改到别处。
依赖 `rsvg-convert`（`brew install librsvg`）渲染 PNG；缺失时只产出 SVG。
